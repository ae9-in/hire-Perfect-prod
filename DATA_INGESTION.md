# MCQ Data Ingestion Pipeline

Use this pipeline for all large MCQ imports:

1. `raw-docx/` - original source files.
2. `raw-pdf/` - original PDF source files.
3. `processed-json/` - parsed JSON output from source file.
3. `validated-json/` - clean JSON generated by validation step.
4. `import-logs/` - audit logs for each import run.

## Required flow

`SOURCE FILE -> PARSE -> VALIDATE -> PREVIEW -> IMPORT -> VERIFY`

Do not insert raw parsed content directly into MongoDB.

## One-command DOCX/PDF pipeline

Use:

- Preview only (recommended first):
  `npm run import:questions:file -- --file=raw-docx/your-file.docx --assessmentId=<ASSESSMENT_ID> --mode=preview`
- Validate only:
  `npm run import:questions:file -- --file=raw-docx/your-file.docx --assessmentId=<ASSESSMENT_ID> --mode=validate`
- Parse + import:
  `npm run import:questions:file -- --file=raw-docx/your-file.docx --assessmentId=<ASSESSMENT_ID> --mode=import`
- PDF preview:
  `npm run import:questions:file -- --file=raw-pdf/your-file.pdf --assessmentId=<ASSESSMENT_ID> --mode=preview`

If one file contains multiple assessments in one category, use category mapping:

- `--categorySlug=<CATEGORY_SLUG>` instead of `--assessmentId`
- each question row should include `subtopic` matching an assessment title in that category

Example:

`npm run import:questions:file -- --file=raw-pdf/1_Generative_AI_for_Business_Leaders.pdf --categorySlug=generative-ai-business-leaders --mode=preview`

Then import:

`npm run import:questions:file -- --file=raw-pdf/1_Generative_AI_for_Business_Leaders.pdf --categorySlug=generative-ai-business-leaders --mode=import`

What it does:

1. Parses DOCX/PDF to `processed-json/<file>.json`
2. Runs validation/preview/import using the existing secure importer
3. Writes `validated-json/*.validated.json` and `import-logs/import-log-*.json`

## JSON input shape

```json
[
  {
    "assessmentId": "65f2f8a5d1d91234567890ab",
    "subtopic": "Python Bootcamp",
    "level": "Entry",
    "question": "Python is a?",
    "options": ["Low-level language", "High-level language", "Database", "Browser"],
    "correctAnswer": 1,
    "explanation": "",
    "tags": ["python", "basics"]
  }
]
```

## Import script

`Backend/scripts/import-questions.ts`

Commands:

- Validate only:
  `npx tsx Backend/scripts/import-questions.ts --mode=validate --file=processed-json/software-development-mcqs.json --assessmentId=<ASSESSMENT_ID>`
- Preview (shows rows, no DB insert):
  `npx tsx Backend/scripts/import-questions.ts --mode=preview --file=processed-json/software-development-mcqs.json --assessmentId=<ASSESSMENT_ID>`
- Import (validated + deduplicated):
  `npx tsx Backend/scripts/import-questions.ts --mode=import --file=processed-json/software-development-mcqs.json --assessmentId=<ASSESSMENT_ID>`

Each run writes:

- validated rows to `validated-json/*.validated.json`
- import log to `import-logs/import-log-*.json`

## Post-import verification queries

```javascript
db.questions.find({ type: "mcq", options: { $not: { $size: 4 } } });
db.questions.find({ type: "mcq", correctAnswer: { $nin: [0, 1, 2, 3] } });
db.questions.aggregate([
  { $group: { _id: { assessment: "$assessment", question: "$normalizedQuestion" }, count: { $sum: 1 } } },
  { $match: { count: { $gt: 1 } } }
]);
```
